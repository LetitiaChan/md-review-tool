## MD Human Review 编辑模式设计方案

### 一、架构总览

编辑模式基于浏览器原生 **`contentEditable`** 实现所见即所得（WYSIWYG）编辑，核心设计理念是**"快照对比 + 增量转换"**——尽可能保留原始 Markdown 文本，仅对用户实际修改的部分做 HTML→Markdown 转换，从而最大程度避免格式损失。

### 二、双模式切换

| 项目 | 预览模式 (preview) | 编辑模式 (edit) |
|------|-------------------|----------------|
| `contentEditable` | `false` | `true` |
| 批注系统 | 激活 | 停用 |
| 数学公式 | KaTeX 渲染 | 还原为 `$...$` 原始文本 |
| 图表 | 渲染为 SVG/img | 转为可编辑 textarea |
| WYSIWYG 工具栏 | 隐藏 | 显示 |
| 切换快捷键 | `Ctrl+Shift+E` | `Ctrl+Shift+E` |

**切换流程** (`switchMode()`, `app.js:1666`)：

**进入编辑模式：**
1. 重新渲染纯净内容（无批注标记）
2. `Renderer.restoreMathPlaceholders()` — 将数学公式占位符还原为 `$...$` / `$$...$$`
3. `convertDiagramsToEditable()` — 图表容器转 textarea
4. 启用 `contentEditable = 'true'`
5. **拍快照** — 保存 `_editSnapshotBlocks`（原始 Markdown blocks）+ `_editSnapshotHtmls`（每个 block 的 innerHTML）

**退出编辑模式：**
1. 若有未保存修改 → 先自动保存
2. 关闭 `contentEditable`
3. 用最新 rawMarkdown 重新渲染完整预览（含批注 + KaTeX + Mermaid 等）

### 三、核心保存算法 — Block-Level Diff

这是整个编辑模式中最复杂的部分，位于 `handleSaveMd()` (`app.js:1803`)。采用**三层转换策略**（优先级递减）：

```
┌─────────────────────────────────────────────┐
│  遍历当前 DOM 中所有 .md-block 元素           │
│  ↓                                           │
│  Step 1: Block 对齐                          │
│  通过 HTML 内容匹配建立                       │
│  currentBlock[i] → snapshotBlock[j] 映射     │
│  ↓                                           │
│  Step 2: 逐块对比                            │
│  ├── HTML 未变 → 直接保留原始 Markdown ✅     │
│  ├── 简单文本变 → 行级 diff 精确替换 ✅      │
│  └── 复杂变化 → Turndown 全量转换 ✅         │
│  ↓                                           │
│  Step 3: 组装 → 通过 postMessage 写回源文件   │
└─────────────────────────────────────────────┘
```

**第一层：Block 未变化 → 零损失**
- 对比当前 block 的 innerHTML 与快照 HTML
- 完全相同 → 直接输出原始 Markdown，无任何转换损耗

**第二层：行级 Diff → 精确替换**
- 条件：行数相同、不含复杂结构（blockquote/code/diagram 等）
- 通过 `stripMdMarkers()` 去除 Markdown 标记后对比纯文本
- 文本变了但结构没变 → 保留原始 Markdown 前缀（`#`、`-`、缩进等），只替换文本部分
- 如果原始行含内联格式（`**加粗**`、`` `代码` `` 等）→ 放弃，fallback 到第三层

**第三层：Turndown 全量转换（仅用于变化的 block）**
- 使用 TurndownService + 20+ 自定义规则将 HTML 转回 Markdown

**Block 对齐算法** (`app.js:2345`)：
- 第一遍：精确匹配（HTML 完全相同），优先在 ±3 位置查找
- 第二遍：未匹配的 block 按顺序分配最近的未使用 snapshot

### 四、TurndownService 自定义规则

| 规则名 | 功能 |
|--------|------|
| `blockquote` | `<blockquote>` → `> ` 前缀引用 |
| `ghAlert` | `div.gh-alert` → `> [!NOTE]` / `> [!WARNING]` 等 GitHub 告警语法 |
| `codeBlock` | `div.code-block > pre > code` → ` ```lang ` 围栏代码块 |
| `diagramEditWrapper` | 图表 textarea → ` ```mermaid ` / ` ```plantuml ` / ` ```dot ` |
| `mermaidContainer` | Mermaid 渲染容器 → ` ```mermaid `（从 `data-source` 解码） |
| `plantumlContainer` | PlantUML 容器 → ` ```plantuml ` |
| `graphvizContainer` | Graphviz 容器 → ` ```dot ` |
| `table` | `<table>` → Markdown 表格（保留 cell 内行内格式） |
| `image` | `<img>` → `![alt](src)`，decodeURIComponent 还原中文路径 |
| `highlight` | `<mark>` → `==文本==` |
| `superscript` | `<sup>` → `^文本^`（排除脚注引用） |
| `subscript` | `<sub>` → `~文本~` |
| `strikethrough` | `<del>/<s>` → `~~文本~~` |
| `taskListItem` | `li.task-list-item` → `- [x]` / `- [ ]` |
| `coloredText` | `<span style="color:xxx">` → `{color:xxx}文本{/color}` |

### 五、智能编辑保护

针对编辑模式下容易被 `contentEditable` 破坏的特殊结构：

1. **数学公式**：进入编辑模式时，`restoreMathPlaceholders()` 将 KaTeX 渲染的 HTML 还原为 `$...$` 纯文本，用户直接编辑公式源码
2. **图表**：`convertDiagramsToEditable()` 将 Mermaid/PlantUML/Graphviz 渲染结果替换为 `<textarea contentEditable="false">`，隔离外层 contentEditable 影响
3. **代码块**：`blockHtmlToMarkdown()` 中有专门的 `extractCodeText()` 递归提取器，正确处理浏览器在 contentEditable 中产生的 `<div>/<br>` 换行
4. **引用块/告警块**：在行级 diff 中检测到 `blockquote, .gh-alert` 等复杂结构时，直接跳过行级 diff，使用 Turndown 规则还原

### 六、图表源码编辑

`convertDiagramsToEditable()` (`app.js:1298`)：
- 遍历所有 `.mermaid-container`、`.plantuml-container`、`.graphviz-container`
- 替换为 `div.diagram-edit-wrapper` + `textarea.diagram-edit-textarea`
- textarea 设置 `contentEditable = 'false'`（阻止外层 contentEditable 影响）
- 支持 Tab 键缩进（4 空格）
- 自动高度调整 (`autoResizeTextarea`)
- textarea 的 input 事件触发 dirty 标记和自动保存

### 七、表格编辑

右键表格单元格弹出上下文菜单 (`setupTableContextMenu()`, `app.js:1542`)：
- 在上/下方插入行
- 在左/右侧插入列
- 删除当前行/列（保护最后一行/一列不可删）

### 八、WYSIWYG 工具栏

位于 `index.html:169`，使用浏览器原生 `document.execCommand()` API：

| 按钮 | 命令 |
|------|------|
| **B** / *I* / ~~S~~ | `bold` / `italic` / `strikeThrough` |
| H1 H2 H3 | `formatBlock` → `<H1>` / `<H2>` / `<H3>` |
| 无序列表 / 有序列表 | `insertUnorderedList` / `insertOrderedList` |
| 引用块 / 分隔线 | `formatBlock → blockquote` / `insertHorizontalRule` |
| 撤销 / 重做 | `undo` / `redo`（浏览器原生撤销栈） |

### 九、保存机制

| 触发方式 | 说明 |
|----------|------|
| `Ctrl+S` | 手动保存 |
| 自动保存 | 编辑后 1.5 秒无操作自动触发 (`scheduleAutoSave`) |
| 切换回预览 | 自动保存未保存的修改 |
| 切换文件 | 自动保存后再切换 |
| 一键 AI 修复 | 先自动保存再生成指令 |

保存路径：`Webview → postMessage('saveFile') → reviewPanel.ts → fileService.saveFile() → 写入磁盘`

保存成功后**不重新渲染 DOM**（避免破坏编辑状态），而是更新内部快照：
```js
_editSnapshotBlocks = Renderer.getRawBlocksBeforeExtract().slice();
_editSnapshotHtmls = Array.from(docContent.querySelectorAll('.md-block')).map(el => el.innerHTML);
```

### 十、状态管理

- `editorDirty` — 是否有未保存修改
- `updateEditStatus(className, text)` — 显示状态（编辑模式 / 未保存 / 保存中 / 已保存 / 保存失败）
- Task Checkbox 直接在 `rawMarkdown` 中精确修改对应行，不走 Turndown（`app.js:270`）
